Reference: General Facts



Reference is a description of all functions of HP MIDIFILE. For each group of functions the topics "General fact" and "Samples" are presented.


The denotation of MIDI-channels, MIDI-file-tracks, voice-, takt- and beat-numbers

With a lot of function we need to specify a number for a MIDI-channel or a track or a number for a voice. Sequencer-software most numbers the MIDI-channels and tracks starting from 1. The method of numbering voices differs: some start with 0, others with 1. Similarly with takt- and beat-numbers. In sequencer software the first takt and the first beat of a takt have the number 1.

HP MIDIFILE consequently uses the internal numbering, which starts with 0. I.e.

0 <= chan <= 15 with MIDI channels
0 <= track with tracks
0 <= voice <= 127 for voices
0 <= takt with the numbers of measures
0 <= beat < num with the beats of a measure

The structure of MIDI-files of type SMF1, created by HP MIDIFILE.

The MIDI-file will be created by the function HP_Save. Two MIDI-file-formats are possible: SMF0 or SMF1.

SMF1 files get a fixed structure which is independed from the original SMF1-file, which is loaded with HP_Load. The SMF1 MIDI-file gets exactly 17 tracks. The first track (no. 0) is filled with all MIDI-events which are not associtated with a MIDI-channel. These are SysEx and most Meta-Events. The following 16 tracks are connected with MIDI-channels: track 1 for channel 0, track 2 for channel 1, ... track 16 for channel 15. HP MIDIFILE does not use any trackname of the original MIDI-file, but it is allowed to create new ones.
