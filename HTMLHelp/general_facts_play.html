Play: General Facts



HP MIDIFILE offers functions to play MIDI events of the actual MIDI-file session on all MIDI output devices of the soundcard. The first action is to find out the right MIDI device ID. The function HP_GetMIDIDevices creates a list of all ID's together with its names. After this the wanted device must be set with the function HP_SelectMIDIDevice, which is selected global for all MIDI-file sessions. If you want to select the Microsoft MIDI-Mapper always, you need not to use these function; the inital ID is connected to this device.

The function HP_Play plays all or all selected MIDI events of a MIDI-file session. If the selection does starts with time > 0; the user can specify, that all SysEx-, Controller- and Program-Change-events of earlier times will be send before in order to get the correct voice-effects.

HP_Play does not send any System-Common-Events like MIDI Start, MIDI Clock or MIDI Stop to synchronise a slave MIDI device.

It is not possible, to finish a MIDI-file session with HP_Free, if a play is still running.

HP MIDIFILE applies the functions of the "Platform SDK: Windows Multimedia" and the timer of the IReferenceClock Interface.

HP MIDIFILE offers an interface for monitoring takt and beat, lyrics and YAMAHA chords while playing.
