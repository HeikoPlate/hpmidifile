Stream Access: General Facts



During a MIDI-file session HP MIDIFILE helds all MIDI-events in a dynamic array, the eventlist. The events are sorted in increasing order by its time-stamps.

HP MIDIFILE offers functions which handles the eventlist as stream. Reading an event with the function HP_ReadEvent, we get back an "id", i.e. an index of the array eventlist. This "id" is used with a lot of functions to identify the event.

It is allowed always to continue reading sequentially even if events are deleted or inserted into the eventlist, but we will not get a new id, which is increased by one from the last all time. Therefore it is not recommended to perform arithmetics with id's.
