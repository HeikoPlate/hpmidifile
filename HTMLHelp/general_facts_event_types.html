Event Types: General Facts



The MIDI-events are organisized in HP MIDIFILE with a tag, which describes the event types. The types are "unsigned int" (UINT) constants and in many functions these are used with the parameter "type".

The table of event-types in Types is divided in three columns: The name of the UINT type, the category, and the meaning.


The categories

Standard Standard MIDI-events, status from 0x8? to 0xe?, without controller-events but with NRPN and RPN
Controller Standard Controller-events, status 0xb?, but without NRPN and RPN
SysEx Systeme Exclusive events, status 0xf0
Standard Meta Standard Meta-events, from 0xff-0x00 to 0xff-0x59
YAMAHA Meta Meta-events, beginning 0xff-0x7f-0x43
Event User defined event
Meta Unknown Meta-event


The event-types HP_NOTE, HP_RPN and HP_NRPN make sense only as sequence of more than one elementary MIDI-event, we denote these as "Compount MIDI-events". For example HP_NOTE consists of the couple "Note On" and "Note Off"; RPN / NRPN of more ones. The user of HP MIDIFILE is not allowed to work with these special elementary events, but he is able to read events of these types. In the column "meaning" the comment "Hidden" is added here.

In the MIDI specification Controller events have only one event-type but different functions, which are specified with a number of the Controller event. HP_MIDIFILE here spends for each Controller-number an own event-type. This method allows to identify the events more easy.

Testing an application the function HP_TypeName, which returns the string of the type, may be helpful.
