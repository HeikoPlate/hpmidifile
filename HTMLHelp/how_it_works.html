How it works



Overview

HP MIDIFILE consists of a lot of functions to manipulate existing MIDI-files or to create new ones. The functions operate on global attributes of the MIDI-file and on MIDI-events.

The data of a MIDI-file is internal organisized by an dynamic eventlist of MIDI-events which is sorted by time-stamps. The function HP_ReadEvent reads the MIDI-events sequentially from the eventlist and delivers the time-stamp and a tag, describing the type of the event.

We distinguish standard MIDI-events, SysEx-events, standard Meta-events and YAMAHA meta-events, used with the formats XG and XF. For most events read-, insert- and change-functions are implemented.

Other functions allow to shift events, to transpose and quantisize notes, to select, copy and paste and to play the file.

The first action to be done is to initialize a MIDI-file session with the call HP_Init. The function returns a handle, which now is the first parameter for all function-calls of this session. We can set up more than one session at the same time and we can copy events from one session to another.

The next call will concern mostly the function HP_Load in order to load an existing MIDI-file of type SMF0 or SMF1. But it is also possible to edit a new MIDI-file by entering all events needed without loading a MIDI-file before. The creation of the target MIDI-file happens with the call of HP_Save. Here the formats SMF0 and SMF1 are allowed. The session ends with a call of HP_Free, which destroys all data from memory.

Additional a set of functions is included to get texts from Instrument Definition Files of Cakewalk/Sonar.
